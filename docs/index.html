What Douglas does

DFIR analysts have a folder full of triage CSVs and want to triage them quickly. Douglas opens the case, parses every artifact, and gives you per-host context.

📂

Opens a case directory

Point Douglas at the output of Run-ZimmermanTools.ps1 (or any case folder with the same structure). It discovers hosts, parses CSVs, derives identity / hardware / network from the registry hives, and surfaces empty artifacts so you know what was actually collected.

🦅

Surfaces detections first

Hayabusa rows are categorised by severity at parse time. The host overview shows a histogram per severity bucket so you can pick the host that needs attention before clicking anything. MPLog and EvtxECmd alerts feed the same view.

🚩

Marks per row

Flag a row from any artifact as suspicious. Marks are scoped to the case folder (no external database). The titlebar shows the count; the timeline view aggregates them across hosts so you can build a case narrative without leaving the app.

⚙

Built-in preprocessor wizard

Run Run-ZimmermanTools.ps1 from inside Douglas. PowerShell discovery, live log streaming, cancel button. Embedded PS1, no separate file to lose.

⌛

Time-window correlation

Click any row with a timestamp; Douglas shows everything from other artifacts within a ±N-second window. Great for chaining Sysmon process creation to its prefetch/amcache trail.

🔒

Local-only, single binary

Pure Go, zero external deps. Binds to a random loopback port. CSRF + Host-header allowlist + CSP enforced. No telemetry, no auto-update, no cloud.

Install

Requires Go 1.21+. Single static binary, no external libraries to vendor.

Build from source

Clone the repo and build for your platform.

# Clone + build
git clone https://github.com/yourname/douglas
cd douglas
make build

# Or build for Windows from elsewhere
GOOS=windows GOARCH=amd64 go build \
  -ldflags "-s -w" \
  -o douglas.exe \
  ./cmd/artifact-review

Run

Point at a case folder. Browser opens automatically.

# Open an existing case
./douglas --case /path/to/case

# Or run with no case and use the wizard to create one
./douglas

# Common flags
--case <dir>         # open case on launch
--addr 127.0.0.1:0   # random port (default)
--no-open            # don't auto-open browser
--upload-workers 2   # parallel job slots
